GOAL:
Build UI system for unified SEO platform using layered architecture, not screen-by-screen design.

---

FOUNDATIONAL PRINCIPLE:

UI must be built as a system composed of:
- archetypes
- behaviour anchors
- patterns
- design language
- design system
- component layer
- SVG icon layer
- iteration / polish passes

Freeze each layer before moving to the next.

---

PHASE 1 — ARCHETYPE EXTRACTION

DEFINE CORE SCREEN ARCHETYPES:

- dashboard
- keyword screen
- rank screen
- content screen
- review screen
- competitor screen
- settings

PURPOSE:
Reduce product surface into reusable structural screen types.

---

PHASE 2 — BEHAVIOUR OVERLAY

ATTACH TO EACH ARCHETYPE:

- user intent
- primary actions
- secondary actions
- states
- state transitions

PURPOSE:
Anchor structure to behavioural truth before styling.

---

PHASE 3 — MARKET OVERLAY

APPLY:

- SEO conventions
- ASO conventions
- user expectations
- trust cues
- category norms

PURPOSE:
Ensure system aligns with market expectations.

---

PHASE 4 — INSPIRATION SCREEN PATTERN EXTRACTION

EXTRACT FROM REFERENCE SCREENS:

- layout patterns
- navigation patterns
- table patterns
- chart patterns
- card patterns
- filter patterns
- insight / alert patterns

PURPOSE:
Extract proven screen patterns instead of inventing ad hoc UI.

---

PHASE 5 — PATTERN EXTRACTION CHECKLIST

FOR EACH EXTRACTED PATTERN DEFINE:

- pattern name
- purpose
- where used
- structure
- behaviour notes
- allowed variants
- anti-patterns

PURPOSE:
Formalize pattern extraction into reusable governance.

---

PHASE 6 — PATTERN LIBRARY

BUILD REUSABLE PATTERN LIBRARY FOR:

- keyword tables
- ranking charts
- review clusters
- competitor comparison blocks
- insight cards
- filters
- action panels
- alerts / notifications

PURPOSE:
Create reusable pattern inventory before visual assembly.

---

PHASE 7 — DESIGN LANGUAGE

DEFINE VISUAL LANGUAGE FOR THE SYSTEM:

- visual tone
- density level
- hierarchy style
- spacing character
- card treatment
- border/radius style
- shadow/elevation style
- icon style
- data-display feel
- trust/premium cues

PURPOSE:
Create the visual expression layer that sits above structure/patterns.

NOTE:
This is design language, not component implementation.

---

PHASE 8 — DESIGN SYSTEM

DEFINE SYSTEM RULES FOR:

- typography
- spacing
- color
- layout
- sizing
- elevation
- interaction states

PURPOSE:
Convert design language into enforceable reusable rules.

---

PHASE 9 — COMPONENT-BASED SVG ICON LAYER

BUILD ICON LAYER AS A SYSTEM:

- SVG icon set
- component-based usage
- consistent stroke/fill logic
- consistent sizing behavior
- reusable icon wrappers for UI components

PURPOSE:
Ensure iconography is systematic, reusable, and aligned to design language.

NOTE:
This layer is separate from general components because it is a foundational visual system of its own.

---

PHASE 10 — COMPONENT SYSTEM

BUILD REUSABLE COMPONENTS:

- cards
- tables
- filters
- charts containers
- insight blocks
- alerts
- buttons
- inputs
- navigation items

PURPOSE:
Translate design system into reusable implementation units.

---

PHASE 11 — ASSEMBLY ON ARCHETYPES

COMBINE:

- archetypes
- behaviour overlays
- market overlays
- extracted patterns
- design language
- design system
- SVG icon layer
- component system

PURPOSE:
Construct screen systems by composition, not one-off design.

---

PHASE 12 — ITERATION LAYERS

APPLY IN ORDER:

PASS 1:
- structural refinement

PASS 2:
- hierarchy refinement

PASS 3:
- design language application

PASS 4:
- design system enforcement

PASS 5:
- density / clarity tuning

PASS 6:
- polish pass

PURPOSE:
Refine one layer at a time without collapsing architecture discipline.

---

PHASE 13 — IMPLEMENTATION

CODEX BUILDS IN ORDER:

- SVG icon layer first
- components second
- archetype screens third
- product screens via composition last

PURPOSE:
Keep implementation aligned with system architecture.

---

RULES:

- do not start from polished screens
- do not skip archetypes
- do not skip behaviour anchors
- do not skip design language
- do not skip SVG icon layer
- freeze each layer before moving forward
