# Frontend Phase Index

## Purpose
This index is the review and execution map for frontend work anchored to `SYSTEMATIC_UI_ARCHITECTURE.md`.

Each phase document exists to:
- define scope
- define inputs
- define required outputs
- define completion criteria
- provide the anchor used for review before moving to the next phase

## Review Rule
- No phase is considered complete until its completion criteria are explicitly checked.
- No downstream phase should be implemented on the assumption that an upstream phase is “close enough”.
- Gap analysis should reference these phase docs first, then the master source docs.
