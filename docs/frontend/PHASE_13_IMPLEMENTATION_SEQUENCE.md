# Phase 13: Implementation Sequence

## Objective
Keep code implementation aligned with the systematic architecture instead of drifting into ad hoc screen work.

## Required Build Order
1. SVG icon layer
2. components
3. archetype screens
4. product screens via composition

## Execution Rules
- do not start from polished screens
- do not skip archetypes
- do not skip behaviour anchors
- do not skip design language
- do not skip SVG icon layer
- freeze each layer before moving forward

## Frontend Review Use
This document is the execution gate used before implementation work begins on each downstream stage.

## Completion Criteria
- the build order is reflected in the actual frontend codebase
- foundational layers are present before product-level refinement begins
- implementation sequencing can be defended against the roadmap line by line

## Backfill Status
- historical build-order drift has been backfilled
- icon layer, shared components, archetype scaffolds, gated modules, and operational flow framing now exist before any refinement passes
- later work should be evaluated against the corrected architecture, not against the earlier partial MVP shell
